# v0.4.0-beta.4 — Release QA evidence

Run the release QA matrix and attach the output directory as release evidence.

```bash
cd hardproof
./scripts/release/qa_matrix.sh
```

Cases:

- `stdio-filesystem`: `npx -y @modelcontextprotocol/server-filesystem /tmp`
- `stdio-github`: `npx -y @modelcontextprotocol/server-github` (may require a GitHub token, depending on the server’s config)
- `http-good`: `http://127.0.0.1:18080/mcp` (Hardproof’s `good-http` fixture)
- `http-trust-full`: `http-good` plus `--server-json` and `--mcpb` trust inputs

Expected outputs per case (under `out/release-qa/<timestamp>/<case>/`):

- `scan.json`
- `scan.events.jsonl`
- `report.summary.txt`
- `report.html`
- `report.sarif.json`
